接口请求约定

1. 服务器请求接口地址

  • 开发服务器 https://api.xxx.com

2. 请求的方式

  • 所有的请求都以 RESTful API 的方式发送
    RESTful API风格参考一
    RESTful API风格参考二
    RESTful API风格参考三
  • 通过 http 请求类型区分操作类型
  • GET - 阅读资源
  • POST - 创建新资源
  • PUT - 修改现有资源
  • DELETE - 删除资源
  • 根据 http 状态码返回请求状态
  • 200 OK 成功的要求
  • 201 Created 保存新对象
  • 204 No content 对象已删除
  • 400 Bad Request 返回带有错误消息的JSON
  • 401 Unauthorized 无法验证您的请求
  • 402 2FA Token required使用用户的2FA令牌重新尝试请求作为2FA-Token标头
  • 403 Invalid scope 用户未授权必要的范围
  • 404 Not Found 没有这样的对象
  • 429 Too Many Requests 您的连接速度有限
  • 500 Internal Server Error 出了些问题
  • 503 Service Unavailable 您的连接受到限制或服务因维护而中断

3. 验证请求的有效性

  • 采用 Token 的方式验证用户登陆的有效性，由服务器判断用户的有效性；Token 在服务器端应当是有时效性的，过期需要刷新 Token，刷新 Token 失败应重新调用登录接口获取 Token
  • Token 采用 JWT 的方式进行鉴权

4. 调用频率限制

  • 为了防止服务器被攻击，减少服务器压力，需要对接口进行合适的限流控制，服务器在响应头信息中加入限流的信息，告知客户端当前的限流情况
  • 返回 Headers 信息
  • 超过限流次数后，服务器会返回 429 Too Many Requests 错误。

5. 接口版本控制与数据格式

  • 可以通过添加头信息 Accept 访问不同接口版本与数据格式
